# AI Node Onboarding API Contract

Status: Planned
Implementation status: Not developed
Last updated: 2026-03-11

## Purpose

This document defines the canonical API contract for node-initiated onboarding session creation.
This endpoint creates onboarding intent only and does not issue trust credentials.

## Endpoint

Status: Planned

- Method: `POST`
- Path: `/api/system/nodes/onboarding/sessions`
- Auth mode: onboarding bootstrap (no trusted node token required yet)
- Contract version field: request includes `protocol_version`; Core validates support.

## Request Schema

Status: Planned

```json
{
  "node_name": "office-node-1",
  "node_type": "ai-node",
  "node_software_version": "0.1.0",
  "protocol_version": "1.0",
  "hostname": "office-node-host",
  "node_nonce": "b7f4c6b4-7787-4cbc-88ea-7f9a6f4f9d04"
}
```

Required fields:
- `node_name` (string, non-empty)
- `node_type` (string, must equal `ai-node`)
- `node_software_version` (string, non-empty)
- `protocol_version` (string, non-empty; must be supported by Core)
- `node_nonce` (string, non-empty; binding value generated by node)

Optional fields:
- `hostname` (string)

## Success Response Schema

Status: Planned

```json
{
  "ok": true,
  "session": {
    "session_id": "sx_onb_8Fjlx7...",
    "onboarding_status": "pending_approval",
    "approval_url": "https://core.example.com/onboarding/approve?sid=sx_onb_8Fjlx7...&state=...",
    "expires_at": "2026-03-11T18:45:00+00:00",
    "finalize": {
      "method": "GET",
      "path": "/api/system/nodes/onboarding/sessions/sx_onb_8Fjlx7.../finalize"
    }
  }
}
```

Required response fields:
- `session.session_id`
- `session.onboarding_status` (must be `pending_approval`)
- `session.approval_url`
- `session.expires_at`
- `session.finalize.method`
- `session.finalize.path`

## Error Contract

Status: Planned

Error shape:

```json
{
  "detail": {
    "error": "error_code",
    "message": "Human-readable explanation",
    "retryable": false
  }
}
```

Canonical validation and policy errors:
- `malformed_request` (`400`)
- `node_type_unsupported` (`400`)
- `protocol_version_unsupported` (`400`)
- `duplicate_active_session` (`409`)
- `registration_disabled` (`503`)

Recommended additional errors:
- `node_nonce_required` (`400`)
- `node_name_required` (`400`)

## Deterministic Behavior Rules

Status: Planned

- Endpoint creates onboarding session intent only.
- Endpoint must not issue trust token or MQTT operational credentials.
- Duplicate active onboarding sessions for the same node binding must return `duplicate_active_session`.
- Session expiry must be explicit and returned as `expires_at`.
- Response fields must be stable for node-side parsing and versioned evolution.

## Versioning Rules

Status: Planned

- `protocol_version` is required in request.
- Core may support a set of protocol versions and reject unsupported versions.
- Backward-incompatible contract changes require new protocol version support path.
